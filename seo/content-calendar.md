## Post Structure Template

Each blog post should follow this structure:

1. **Title** — Keyword-rich, under 60 characters if possible
2. **Meta description** — 150-160 characters, includes primary keyword
3. **Introduction** (100-200 words) — Hook, state the problem, preview the solution
4. **Body** (800-1,500 words) — H2 and H3 sections, include comparison tables where relevant
5. **Conclusion** (100-200 words) — Summary, CTA to try Novyx Vault
6. **Internal links** — Link to at least 2 other blog posts + /features page
7. **Images** — At least 1 screenshot or diagram per post, with descriptive alt text

## Differentiation Angles

Novyx Vault's unique differentiator is persistent AI memory via the Novyx SDK. Every post should naturally highlight at least one of:

- AI memory that evolves over time (vs stateless AI in competitors)
- Memory rollback and transparency
- Ghost Connections (AI-discovered note relationships)
- Local-first privacy with optional cloud
- Bring your own AI key (provider agnostic)
- Open source and self-hostable
